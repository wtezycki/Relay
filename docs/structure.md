# structure.md - Project Architecture

This document defines the mandatory directory structure for the Relay project. Every new file or module generated by the AI agent must strictly follow this organization.

## Root Directory Overview
```text
relay/
├── backend/            # Spring Boot (Java) - REST API & Strava Sync Logic
├── frontend/           # React (Vite) - Main User Interface & Dashboard
├── docs/               # System Prompts & Guidelines (memory.md, Skill.md, structure.md)
└── .gitignore          # Root-level ignore rules
```

## Backend Structure
```text
backend/
├── src/
│   ├── main/
│   │   ├── java/com/relay/
│   │   │   ├── core/           # Cross-cutting: Security, Config, Exceptions, Utils
│   │   │   ├── user/           # User domain (Strava Auth, Profile, Streaks)
│   │   │   │   ├── UserController.java
│   │   │   │   ├── UserService.java
│   │   │   │   ├── UserRepository.java
│   │   │   │   ├── User.java (Entity)
│   │   │   │   └── UserDto.java
│   │   │   ├── challenge/      # Challenge domain (Global Progress, Milestones)
│   │   │   ├── activity/       # Activity domain (Strava Sync, Activity Normalizer)
│   │   │   │   ├── ActivitySyncWorker.java (Cron Scheduler)
│   │   │   │   ├── ActivityNormalizerService.java
│   │   │   │   └── ...
│   │   │   ├── feed/           # Social domain (Live Team Feed, Kudos)
│   │   │   └── RelayApplication.java # Main entry point
│   │   └── resources/
│   │       ├── application.yml # App config (DB, Strava OAuth credentials)
│   │       └── db/             # SQLite schema or migrations
├── pom.xml                     # Maven dependencies
└── mvnw                        # Maven wrapper
```

## Frontend Structure
```text
frontend/
├── src/
│   ├── assets/         # UI assets, icons, SVGs
│   ├── components/     # Reusable UI components (ProgressBar, ActivityCard, Feed)
│   ├── context/        # React Context API (Global State: AuthContext, ChallengeContext)
│   ├── hooks/          # Custom hooks (e.g., useStravaSync, useChallengeProgress)
│   ├── pages/          # Full page views (Dashboard, Leaderboard)
│   ├── services/       # API communication (Axios instances)
│   ├── App.jsx         # Root component
│   └── main.jsx        # Entry point
├── tailwind.config.js  # Tailwind CSS configuration
└── package.json
```
